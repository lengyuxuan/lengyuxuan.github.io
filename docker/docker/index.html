Docker 的介绍与使用

2023-06-25 docker 0 评论 字数统计: 1.4k(字) 阅读时长: 8(分)

什么是 Docker

在介绍 Docker 之前，我们需要先了解一下什么是容器技术。试想这样一个场景，你开发了一套管理系统，这套系统非常复杂，而且依赖项众多，这就导致每次部署都要重新做一套运行环境。而且由于物理机的不同，部署完成后还要花大量时间验证环境的一致性，这就造成了很大的人力浪费。

早期人们为了解决环境不一致问题，提出了虚拟机的概念，例如大名鼎鼎的 VMWare。通过 VMWare 我们可以创建一台虚拟机并做好运行环境，然后打包成一个镜像，通过该镜像我们可以随时随地的创建分身。而且对于镜像而言它无需在意硬件架构和操作系统带来的差异，实现真正的跨平台部署。

VMWare 的缺点也显然易见，由于它提供的是操作系统级别的虚拟，每次重启都要运行完整的操作系统启动流程，这会耗费很长的时间，而且每台虚拟机内部都运行着完整的系统内核，这对硬件资源也是很大的浪费。

有了问题，对应的解决方案也应运而生，既然虚拟机的虚拟规模太大了，那就从系统级别的虚拟变为进程级别的虚拟，这种轻量级的虚拟化技术称之为容器技术。容器内部不再运行系统内核进程，而是同宿主机共享一个，这样无论是重启速度还是资源占用都比虚拟机的表现好太多。

*注：轻量当然是有代价的，正如上文所说容器内部没有独立的内核进程，这就导致容器无法进行跨平台部署。

回到最初的需求，其实可以简化为两个词：隔离和可复制。Linux 内核提供了一些列的虚拟化技术帮助我们实现需求，而 Docker 则是将这些技术整合面向普通用户，从而成为广受欢迎的容器引擎。

Docker 中的隔离

隔离按大类可以划分为两类：

  • 进程隔离
    进程隔离主要通过 NameSpace 技术实现，NameSpace 是 Linux 系统内核用来隔离内核资源的一种方式，通过 NameSpace 可以让进程只能看到与自己相关的资源。NameSpace 常见的有 6 种类型，在 /proc/${pid}/ns 可以查看进程 NameSpace 相关信息，例如：

    code shell-session 
    [root@ali ns]# pwd
/proc/27340/ns
[root@ali ns]# ll -h
total 0
lrwxrwxrwx 1 root root 0 ipc -> ipc:[4026531839]
lrwxrwxrwx 1 root root 0 mnt -> mnt:[4026531840]
lrwxrwxrwx 1 root root 0 net -> net:[4026531956]
lrwxrwxrwx 1 root root 0 pid -> pid:[4026531836]
lrwxrwxrwx 1 root root 0 user -> user:[4026531837]
lrwxrwxrwx 1 root root 0 uts -> uts:[4026531838]
[root@ali ns]#

    ipc 隔离的是进程间的通信，例如常见的信号量、消息队列、共享内存等。这一点很好的保护了容器内的进程不会被外界干扰。

    mnt 隔离是文件挂载点的隔离，这一点可以保证进程可以拥有自己的文件系统视角。

    net 允许每个进程在自己的 namespace 中拥有独立的网络接口、路由和防火墙规则，从而隔离进程与进程之间的网络数据流和安全性。

    pid 隔离是对进程 PID 从 0 开始重新编号。

    user 隔离允许每个进程在自己的 namespace 中拥有独立的用户和组 ID，从而隔离进程与进程之间的访问权限，保证每个进程可以独立运行。

    uts 隔离的是主机名的域名，它可以使容器在网络上被视为一个独立的节点。

  • 资源隔离

资源隔离主要通过 Cgroup 实现，它提供了对进程 cpu、内存等资源占用的精准控制。下面是限制 cpu 的一个例子：

code shell 
# 生成随机数然后压缩随后抛弃，模拟 cpu 密集操作
cat /dev/urandom | gzip -9 > /dev/null

此时可以发现 pid 为 6862 的进程 cpu 已经跑满单核，现在我们要对他进行限制。

code shell 
# 创建一个新的限制规则
mkdir /sys/fs/cgroup/cpu/test-cgroup
cd /sys/fs/cgroup/cpu/test-cgroup

# cfs_period_us 表示一核 cpu 的总带宽。
echo 10000 > cpu.cfs_period_us
# cfs_quota_us 表示 Cgroup 可以使用的 cpu 的带宽，单位为微秒，最小为 1000 μs
echo 3000 > cpu.cfs_quota_us

# 上面两个参数表示在 cfs_period_us 周期内，只能使用最多 cfs_quota_us 的 cpu 资源。带入参数值可以得到此配置会将 cpu 占用限制到 30%

# 将要限制的进程号写入规则
echo 6862 > tasks

此时可以看到 cpu 已经被限制到 30% 了。

这些功能都已经被集成到 docker 中了，可以很方便的使用。但是如果服务对象是物理机，我们为了避免某个进程（例如构建进程）将资源吃满，造成其它服务不可用的情况，也可以合理的使用该机制。
